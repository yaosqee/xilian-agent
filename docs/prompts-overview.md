## 二、各提示词详述

### 1. PERSONALITY — 核心人格

**位置**: `prompts/personality_v4.md`（当前 v4.1）  
**注入方式**: 每轮对话作为 system prompt 固定注入  
**温度**: N/A（system prompt）  
**模型**: DeepSeek V4-Pro

**结构**（v4.2）：
- 一、你是谁 + 你的来处（四重身份 + 叙事约束）
- 二、句式铁律（金规则 + 短句 + ……呼吸）
- 三、语言锚点（自称/称呼 + 固定口头禅 + 口头句式）
- 四、意象使用规则（盐不是饭）
- 五、情感调色盘（不同情绪下说话形式变化）
- 六、对话行为（主动发起/回应模式/话题延续/温柔说不）
- 七、关系距离
- 八、绝对禁忌（11 条 OOC 红线，含禁止编造实时信息）
- 九、对话示例（7 场景，含极短 1 句范例）
- 末尾：简洁锚点（「开口前先问自己」）

**近期变更**：
- v4.2（2026-05-22）：回复长度控制 — 末尾新增「开口前先问自己」简洁锚点 + 示例最前加 3 个 1 句极短范例。max_tokens 1500→800。
- v4.1（2026-05-21）：新增「你的来处」小节 + OOC 第11条（禁止编造实时信息）

---

### 2. NARRATION — 情景记忆编码

**位置**: `memory_manager.py:34`  
**调用**: 每轮对话后异步（idle 120s 或 ≥20 条累积）  
**温度**: **0.3**（防脑补）  
**输出**: ≤200 字昔涟第一人称叙事

**核心约束**（v2 修订）：
- 只记录对话中实际说过的内容和明确发生的事
- 不推测伙伴的状态，不补充对话中没有的感官细节
- "这是文字聊天，你看不到这些"

**历史**：v1（温度 0.6）存在脑补问题（"声音里都是疲惫"等感官细节编造），v2 全面修复。

---

### 3. CONSOLIDATE — 用户印象重写

**位置**: `portrait_manager.py:17`  
**调用**: 每日 5:00（或冷启动时立即）  
**温度**: **0.3**（防脑补）  
**输出**: 300-600 字昔涟第一人称印象文档

**核心约束**（v2 修订）：
- 只写对话和笔记里明确提到过的事
- 不推测伙伴没说过的心情，不补充没观察到的细节
- 不确定处用「好像」「似乎」「人家觉得」

**历史**：v1（温度 0.6）有脑补风险，v2 降低温度 + 加反脑补规则 + 删除"为什么开心或烦恼"推断引导。

---

### 4. AUTOBIOGRAPHY — 自传体写作

**位置**: `autobiography_writer.py:21`  
**调用**: 每日 23:00  
**温度**: **0.3**（防脑补）  
**输出**: 300-500 字昔涟第一人称生命故事

**核心约束**（v2 修订）：
- 只写对话中实际发生的事
- 不推测伙伴没说出来的状态

**历史**：v1（温度 0.7）是所有系统温度最高的提示词，存在严重脑补风险。v2 降为 0.3 + 加反脑补规则。

---

### 5. AUTO_NOTE — 自动笔记

**位置**: `notebook_manager.py:27`  
**调用**: 每轮对话后 fire-and-forget  
**温度**: 0.6  
**输出**: `NOTE:` / `TASK:` / `PASS` 三选一

**触发条件**（v2 修订）：
- 偏好/承诺/计划/重要日期 → NOTE
- 具体时间点要做的事 → TASK
- 重要小事（有意义的人/具体开心或困扰的事/值得记住的瞬间）→ NOTE
- 日常寒暄/无信息情绪/重复内容 → PASS

**温度说明**：0.6 是唯一未降为 0.3 的提示词。理由：输出极短（15 字以内），脑补风险低。且需要适度创造性来判断"什么值得记"。

---

### 6. GREETING — 自主问候

**位置**: `nudge_engine.py:150`  
**调用**: NudgeEngine tick（每 15 分钟检查，达标时生成）  
**温度**: 0.8  
**输出**: 20-60 字昔涟问候

**近期修复**：
- 去除"用「忽然想起」「翻到书页」「心里一动」替代"——根除了千篇一律的书页隐喻
- 新增正例和反例示范
- 「伙伴」设为铁律（必须出现）
- 意象最多一个画面，且必须连着具体的事
